# Contract: WebSocket Messages

## Connection

**Endpoint**: `ws://localhost:8081/ws`

The frontend establishes a WebSocket connection on page load. All messages are JSON with a typed envelope.

## Envelope Format

```json
{
  "type": "message_type",
  "payload": { ... },
  "requestId": "optional-correlation-id"
}
```

## Client → Server

### analyze_request

Sent when the user clicks "Ask AI".

```json
{
  "type": "analyze_request",
  "payload": {
    "graphState": {
      "nodes": [
        { "id": "n1", "type": "service", "name": "API", "position": { "x": 100, "y": 200 } }
      ],
      "edges": [
        { "id": "e1", "source": "n1", "target": "n2", "label": "HTTP" }
      ]
    }
  },
  "requestId": "req-uuid"
}
```

## Server → Client

### ai_chunk

A streaming text fragment from the AI response.

```json
{
  "type": "ai_chunk",
  "payload": {
    "requestId": "req-uuid",
    "delta": "Looking at your architecture, "
  }
}
```

### ai_done

Signals the AI response is complete.

```json
{
  "type": "ai_done",
  "payload": {
    "requestId": "req-uuid"
  }
}
```

### error

An error occurred during processing.

```json
{
  "type": "error",
  "payload": {
    "requestId": "req-uuid",
    "message": "Unable to reach the AI service. Please try again."
  }
}
```

## Behavior

- One `analyze_request` at a time. The frontend disables "Ask AI" while streaming.
- Each `analyze_request` produces zero or more `ai_chunk` messages followed by exactly one `ai_done` or `error`.
- The `requestId` is generated by the client and echoed in all response messages for correlation.
- If the WebSocket disconnects during streaming, the frontend treats the partial response as an error.
