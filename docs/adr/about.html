DesignPair ADRs

Architecture Decision Records for an interactive architecture canvas with a real-time AI collaborator

About DesignPair

DesignPair is an interactive architecture canvas where you diagram your system by dragging and connecting components, and an AI collaborator engages as you build — asking probing questions, flagging risks, and suggesting improvements. It's a thinking partner for architectural work, not a scoring tool or diagram generator.

About These ADRs

These Architecture Decision Records document the significant technical decisions behind DesignPair. The collection covers the full stack: a TypeScript/React frontend with React Flow for the interactive canvas, a Go backend for WebSocket communication and LLM integration, and the Anthropic Claude API for real-time architectural analysis.

Sample Queries

Try asking questions that explore the decisions behind DesignPair:

  • “Why was React Flow chosen over alternatives like Excalidraw or D3 for the canvas?”
  • “What are the tradeoffs of using WebSocket for real-time communication versus server-sent events?”
  • “Why was persistence deferred in v1, and what would need to change to add it?”
  • “How does the monorepo structure support independent development of the frontend and backend?”
  • “What drove the decision to use Anthropic Claude rather than a local LLM for the AI collaborator?”

The Bigger Picture

DesignPair is the second portfolio project by Tyler Colbert, alongside ADR Insight (which powers this very page). Together the two projects demonstrate Go, TypeScript/React, two different AI integration patterns (RAG vs. real-time conversational), and a commitment to documenting decisions as they're made.
